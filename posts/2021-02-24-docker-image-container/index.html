이미지

  • 애플리케이션을 실행하는 데 필요한 모든 것을 포함한 소프트웨어 패키지입니다.
    • 코드, 런타임, 시스템 툴, 시스템 라이브러리, 설정
  • 여러 개의 레이어로 구성되어 있습니다.
  • 컨테이너를 생성하고 실행할 때 read-only로 사용됩니다.
  • 이미지의 이름은 다음과 같이 구성됩니다. {저장소}/{이미지}:{태그}
    • (e.g) rabbitmq:3-management
    • 도커 허브의 공식 이미지인 경우 {저장소}를 생략하는 경우도 있습니다.

조회

docker images

컨테이너

  • 이미지로 컨테이너를 생성하면 해당 이미지 목적에 맞는 파일이 들어 있는 독립된 공간이 생성됩니다.
  • 이미지에 변경된 사항만 컨테이너에 저장하므로 원래의 이미지는 영향을 받지 않습니다.
  • 생성된 각 컨테이너는 각자 독립된 파일 시스템을 가지고 있기 때문에 특정 컨테이너가 다른 컨테이너에 영향을 주지 않습니다.

생성

docker run -i -t ubuntu:20.10
  • 컨테이너를 생성 후 실행하고 들어가는 명령어입니다.
  • 이미지가 로컬에 존재하지 않는 경우 도커 허브에서 내려받아 사용합니다.

옵션 설명
-i
-t 
docker create -i -t --name myubuntu ubuntu:20.10
  • 컨테이너를 생성만 합니다.
docker start myubuntu
  • 컨테이너를 실행합니다.
docker attach myubuntu
  • 컨테이너 내부로 들어갑니다.

종료

  • 컨테이너를 정지시키고 나오려면
    • Ctrl + D 또는 exit
  • 컨테이너를 정지시키지 않고 나오려면
    • Ctrl + P, Q
  • 컨테이너 밖에서 정지시키려면 
    docker stop myubuntu

조회

docker ps -a

CONTAINER ID   IMAGE                   COMMAND                  CREATED          STATUS                          PORTS     NAMES
2aa16a831db6   ubuntu:20.10            "/bin/bash"              22 minutes ago   Exited (0) About a minute ago             myubuntu
  • 정지된 컨테이너를 포함한 모든 컨테이너를 출력하려면 -a 옵션을 추가합니다.

삭제

docker rm myubuntu

docker rm -f myubuntu
  • 실행 중인 컨테이너도 삭제하려면 -f 옵션을 추가합니다.
